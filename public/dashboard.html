BACKEND API CONFIGURATION

CyberGuard requires a connection to the backend API server running locally on your computer to perform scans and terminate threat processes.

Connection Checklist:
  • Ensure the backend is running locally (run npm start or node server.js in the project folder).
  • Ensure the port matches the server's listening port (default is 5000).
  • Browser Blockages: If loading CyberGuard over HTTPS (such as on Netlify), browsers like Chrome, Edge, and Firefox exempt localhost from Mixed Content policies, but Safari blocks it entirely. If you have connection failures, try using Chrome/Edge.
